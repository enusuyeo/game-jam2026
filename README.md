## 1. 준비하기

### 1-1. VSCode 설치

아직 VSCode가 없으시다면 아래 링크에서 다운로드하세요.

> **다운로드**: https://code.visualstudio.com/

- **Windows**: `.exe` 파일을 다운로드하여 실행하고, 안내에 따라 설치를 완료하세요.
- **macOS**: `.dmg` 파일을 다운로드하여 열고, VSCode 아이콘을 **Applications** 폴더로 드래그하세요.

### 1-2. Codex 확장 설치

VSCode에서 AI 에이전트(Codex)를 사용하기 위해 확장을 설치합니다.

1. VSCode를 실행합니다
2. 왼쪽 사이드바에서 **확장(Extensions)** 아이콘을 클릭합니다
   - Windows: `Ctrl+Shift+X` / macOS: `Cmd+Shift+X`
3. 검색창에 **"Codex"** 를 입력합니다
4. **OpenAI Codex** 확장을 찾아 **Install** 버튼을 클릭합니다

### 1-3. GameBakery.ai 설치

1. `GameBakery.ai.zip` 파일을 다운로드합니다
2. 게임을 만들 **빈 폴더**에 압축을 해제합니다
   - **Windows**: zip 파일을 우클릭 → **여기에 풀기** (또는 **압축 풀기**)
   - **macOS**: zip 파일을 더블 클릭하면 자동으로 압축이 해제됩니다
3. VSCode에서 해당 폴더를 엽니다
   - **Windows**: 상단 메뉴 **파일 → 폴더 열기**
   - **macOS**: 상단 메뉴 **File → Open Folder**
   - 또는 폴더를 VSCode 아이콘 위에 드래그해도 됩니다

이제 준비가 완료되었습니다!

## 2. 게임 만들기

Codex 에이전트에게 만들고 싶은 게임을 설명하면 됩니다.

### 첫 번째 게임

Codex 채팅창에 다음과 같이 입력해 보세요:

> **"간단한 점프 게임을 만들어 주세요"**

에이전트가 프로젝트 구조를 잡고, 코드를 작성하고, 게임을 완성합니다.

### 이런 것도 가능합니다

- `"우주 슈팅 게임을 만들어 주세요"`
- `"퍼즐 게임인데 블록을 맞추는 거예요"`
- `"캐릭터가 달리면서 장애물을 피하는 게임이요"`
- `"고양이가 생선을 모으는 2D 게임을 만들어 주세요"`

**팁**: 구체적으로 설명할수록 원하는 결과에 가까운 게임이 만들어집니다.

### 뭐부터 해야 할지 모르겠다면?

에이전트에게 전략 조언을 구할 수 있습니다:

> **"어떻게 시작해야 할지 조언해 주세요"** 또는 **"멘토"**

에이전트가 현재 상황을 파악하고, 단계에 맞는 전략을 제안합니다:

- **도구 먼저**: 게임보다 레벨 에디터 같은 도구를 먼저 만들면 생산성이 올라갑니다
- **에셋 검증**: 3D 모델을 바로 만들지 말고, 컨셉 이미지부터 확인 → 모델 → 리깅 순서로
- **MVP 우선**: 최소한의 플레이 가능 버전을 먼저 완성하고, 기능을 하나씩 추가

### 게임 수정하기

게임을 수정하고 싶을 때도 말로 요청하면 됩니다:

- `"점프 높이를 더 높여 주세요"`
- `"배경 색을 파란색으로 바꿔 주세요"`
- `"적 캐릭터를 추가해 주세요"`
- `"점수 시스템을 넣어 주세요"`

Codex에게 다음과 같이 요청하세요:

## 4. 에셋 만들기 (codeb)

codeb는 AI로 게임에 필요한 이미지, 효과음, 3D 모델 등을 생성하는 도구입니다.

### 4-1. codeb 토큰 설정

codeb를 사용하려면 먼저 인증 토큰을 설정해야 합니다.

**토큰 발급:**

1. https://aiproxy.backoffice.bagelgames.com/console/tokens 에 접속합니다
2. 토큰을 발급받습니다

**토큰 등록:**

에이전트에게 다음과 같이 요청하세요:

> **"codeb 토큰 등록해 주세요. 토큰은 aiproxy_xxxx 입니다"**

또는 터미널에서 직접 입력할 수도 있습니다:

```bash
codeb login --token aiproxy_xxxx
codeb whoami   # 인증 상태 확인
```

### 4-2. codeb로 할 수 있는 것

에이전트에게 자연어로 요청하면 됩니다:

| 기능 | 요청 예시 |
|------|----------|
| **이미지 생성** | `"용사 캐릭터 이미지를 만들어 주세요"` |
| **배경 그리기** | `"숲 배경을 그려 주세요"` |
| **배경 제거** | `"이 이미지에서 배경을 제거해 주세요"` |
| **효과음 생성** | `"점프 효과음을 만들어 주세요"` |
| **음성 생성** | `"'게임 오버' 음성을 만들어 주세요"` |
| **3D 모델 생성** | `"검 3D 모델을 만들어 주세요"` |
| **이펙트 시트** | `"폭발 이펙트 스프라이트 시트를 만들어 주세요"` |

### 4-3. 3D 모델 상세

3D 모델은 이미지에서 3D로 변환하는 방식으로 동작합니다.
에이전트가 **단계별로 결과를 확인**하면서 진행합니다:

1. **이미지 생성** → 컨셉이 맞는지 확인
2. **3D 변환** → 브라우저에서 모델 미리보기
3. **리깅** → 뼈대가 잘 들어갔는지 확인
4. **애니메이션** → 동작이 자연스러운지 확인

각 단계에서 마음에 들지 않으면 수정을 요청할 수 있습니다.

사용 가능한 애니메이션: `idle`, `walk`, `run`, `jump`, `slash`

> **"캐릭터 3D 모델을 만들고 walk 애니메이션도 추가해 주세요"**

## 5. 모바일에서 테스트하기

내 폰에서 바로 게임을 테스트할 수 있습니다.
**PC와 같은 Wi-Fi에 연결되어 있어야 합니다.**

### 방법

1. 에이전트에게 **"게임 실행해 주세요"** 라고 요청합니다
2. 로컬 서버가 시작되면 에이전트가 **로컬 IP 주소**를 알려줍니다
   - 예: `http://192.168.0.10:3000`
3. 폰의 브라우저에서 해당 주소를 입력하면 게임을 바로 플레이할 수 있습니다

### 로컬 IP를 모르겠다면

에이전트에게 물어보세요:

> **"내 로컬 IP 주소가 뭐예요?"**

## 6. 모바일 호환성 검사

게임을 모바일(Android)로 배포하기 전에, 호환성 문제가 없는지 검사할 수 있습니다.

> **"모바일 체크해 주세요"** 또는 **"배포 전 점검해 주세요"**

### 검사 항목

| 항목 | 설명 |
|------|------|
| **화면 설정** | viewport 메타 태그가 올바른지 |
| **화면 크기 대응** | 다양한 화면 크기에 맞게 조절되는지 |
| **터치 입력** | 마우스 대신 터치/포인터 입력을 사용하는지 |
| **키보드 전용 조작** | 키보드만으로 조작하는 부분에 터치 대안이 있는지 |
| **소리 재생** | 사용자 조작 없이 자동 재생하는지 |
| **빌드 설정** | playforge 설정 파일이 있는지 |

에이전트가 문제를 발견하면 **구체적인 수정 방법**까지 안내합니다.
검사 후 수정이 필요하면 에이전트에게 **"수정해 주세요"** 라고 요청하면 됩니다.

## 7. 게임 트레일러 만들기

게임 플레이를 녹화하고, AI가 하이라이트를 찾아 앱스토어 프리뷰나 SNS 광고 영상을 자동으로 만들어 줍니다.

> **"게임 트레일러 만들어 주세요"** 또는 **"녹화해 주세요"**

### 과정

1. 에이전트가 브라우저에서 게임을 열고 녹화를 시작합니다
2. 게임을 플레이합니다 (1~3분 권장)
3. "녹화 중지"라고 말하면 AI가 하이라이트를 분석합니다
4. 게임의 세로/가로를 자동 감지하여 맞는 포맷으로 영상을 생성합니다

### 출력물

- **트레일러 영상** (15초 또는 30초) — 앱스토어 프리뷰, TikTok, Instagram용
- **ASO 스크린샷** — 하이라이트 장면에서 자동 추출

> **참고**: FFmpeg가 필요합니다. 없으면 에이전트가 설치를 안내합니다.

## 8. 나만의 스킬 만들기

스킬은 에이전트에게 **새로운 능력**을 추가하는 방법입니다.
반복적으로 하는 작업이나, 특정 규칙을 에이전트가 기억하게 하고 싶을 때 유용합니다.

### 스킬이란?

스킬은 에이전트에게 주는 **설명서**입니다. 예를 들어:

- "레벨 생성 시 항상 이런 규칙을 따라라"
- "UI 디자인은 항상 이 스타일로 해라"
- "특정 API를 이런 방식으로 호출해라"

### 스킬 만들기 (간단한 방법)

에이전트에게 직접 요청하면 됩니다:

> **"새 스킬을 만들어 주세요. 레벨 디자인할 때 항상 적 3마리와 보스 1마리를 배치하는 규칙이에요"**

에이전트가 자동으로 스킬 파일을 생성합니다.

### 스킬 만들기 (직접 작성)

스킬을 직접 작성하고 싶다면 아래 구조를 따르세요.

**1단계: 폴더 만들기**

```
.codex/skills/내-스킬-이름/
└── SKILL.md
```

**2단계: SKILL.md 작성하기**

```markdown
---
name: 내 스킬 이름
description: >-
  이 스킬이 언제 사용되는지 설명합니다.
  "레벨 디자인", "레벨 생성" 같은 요청에 사용합니다.
---

# 내 스킬 이름

## 이 스킬이 하는 일

레벨을 디자인할 때 아래 규칙을 따릅니다.

## 규칙

1. 적은 항상 3마리 배치합니다
2. 보스는 레벨 끝에 1마리 배치합니다
3. 아이템은 중간 지점에 배치합니다
```

**핵심 포인트:**

- `name`: 스킬의 이름입니다
- `description`: **가장 중요합니다!** 에이전트가 이 설명을 보고 "이 스킬을 사용할지 말지"를 판단합니다. 어떤 상황에서 이 스킬이 필요한지 명확하게 적어 주세요.
- 본문: 에이전트가 실제로 따를 지침을 작성합니다

### 스킬 예시 — 게임 난이도 조절

```markdown
---
name: difficulty-balancer
description: >-
  게임 난이도를 조절할 때 사용합니다.
  "밸런스 조절", "난이도 변경", "너무 어려워", "너무 쉬워" 등의 요청에 사용합니다.
---

# 난이도 밸런서

## 규칙

1. Easy: 적 체력 50%, 아이템 드롭률 2배
2. Normal: 기본값
3. Hard: 적 체력 150%, 아이템 드롭률 0.5배

## 적용 방법

- config.js 파일의 DIFFICULTY 상수를 수정합니다
- 적 스폰 로직에서 DIFFICULTY를 참조합니다
```

## 11. 문제 해결

### codeb가 동작하지 않아요

```bash
codeb version    # 설치 확인
codeb whoami     # 인증 확인
codeb upgrade    # 최신 버전으로 업데이트
```

토큰이 등록되지 않았다면 [4-1. codeb 토큰 설정](#4-1-codeb-토큰-설정)을 다시 진행하세요.

에셋 생성 중 에러가 발생했다면 아래 페이지에서 상세 내역을 확인할 수 있습니다:

> https://aiproxy.backoffice.bagelgames.com/errors

### 게임이 실행되지 않아요

- 에이전트에게 `"게임 실행해 주세요"` 라고 다시 요청해 보세요
- `index.html` 파일이 있는지 확인하세요
- 브라우저에서 `http://localhost:3000`에 접속이 되는지 확인하세요

### 모바일에서 접속이 안 돼요

- PC와 폰이 **같은 Wi-Fi**에 연결되어 있는지 확인하세요
- 방화벽이 포트 3000을 차단하고 있을 수 있습니다
  - **Windows**: "Windows 방화벽에서 앱 허용" 알림이 뜨면 **허용**을 클릭하세요
  - **macOS**: "네트워크 연결을 허용하시겠습니까?" 알림이 뜨면 **허용**을 클릭하세요
- 에이전트에게 `"내 로컬 IP 주소가 뭐예요?"` 라고 물어보세요

### APK 빌드가 실패해요

- 인터넷 연결을 확인하세요 (원격 서버에서 빌드됩니다)
- `"모바일 체크해 주세요"` 로 호환성 문제를 먼저 해결하세요
- 에이전트에게 오류 메시지를 보여주고 도움을 요청하세요
