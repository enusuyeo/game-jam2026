# GameBakery.ai Asset Validation

> **목적**: 에셋(특히 3D)을 단계별로 검증하여 크레딧 낭비를 방지하고 품질을 보장한다.

## 핵심 원칙: 바텀업 검증

에셋 생성은 **한번에 완성으로 가지 않는다**. 단계별로 Director에게 결과를 보여주고, 확인받은 뒤 다음 단계로 진행한다. 비용이 큰 단계(3D 변환, 리깅, 애니메이션)는 이전 단계 승인 없이 진행하지 않는다.

## 3D 에셋 검증 순서

```
이미지 생성 → [확인] → 3D 변환 → [프리뷰+확인] → 리깅 → [프리뷰+확인] → 애니메이션 → [프리뷰+확인]
   ~5 크레딧       ~10 크레딧         ~10 크레딧          ~10 크레딧/동작
```

| 단계 | 크레딧 | 검증 내용 | 실패 시 |
|------|--------|-----------|---------|
| 1. 이미지 생성 | ~5 | 컨셉·포즈·배경 확인 | 프롬프트 수정 후 재생성 |
| 2. 3D 변환 | ~10 | 브라우저 프리뷰로 형태 확인 | 이미지부터 재생성 or 옵션 변경 |
| 3. 리깅 | ~10 | 프리뷰로 뼈대 확인 | 리그 타입 변경 후 재시도 |
| 4. 애니메이션 | ~10/동작 | 프리뷰로 동작 확인 | 같은 리그로 재생성 |

## 체크포인트 규칙

1. 각 단계 완료 후 **객관식 질문**(Appendix 3)으로 Director에게 확인
2. 3D 모델/리깅/애니메이션 결과는 **asset-preview 스킬**로 브라우저 프리뷰 제공
3. 다음 단계 진행 전 **추가 크레딧 비용**을 안내
4. Director가 "전부 한번에 해줘"라고 명시하면 일괄 진행 허용 (단, 최종 확인은 필수)

## 이미지 검증 기준

이미지 생성 후 아래 항목을 Director에게 확인:

- **컨셉 일치**: 요청한 캐릭터/오브젝트가 맞는가?
- **포즈 적합성**: 3D 변환에 적합한 포즈인가? (T-pose/A-pose 권장)
- **배경 처리**: 투명/흰색 배경인가? (`--remove-bg` 사용 여부)
- **팔다리 분리**: 캐릭터의 팔다리가 명확히 분리되어 있는가?

## 3D 모델 검증 기준

3D 변환 후 `asset-preview` 스킬로 프리뷰를 열고:

- **형태 충실도**: 원본 이미지와 3D 모델이 닮았는가?
- **폴리곤 수**: 뷰어 정보 패널에서 확인 (웹 기준 8,000 이하 권장)
- **방향**: 정면이 올바른가?
- **텍스처**: 색상과 질감이 자연스러운가?

## 리깅/애니메이션 검증 기준

- **뼈대 구조**: 관절이 올바른 위치에 있는가?
- **동작 자연스러움**: 애니메이션이 부자연스럽게 꺾이지 않는가?
- **루프**: 반복 재생 시 자연스럽게 이어지는가?
