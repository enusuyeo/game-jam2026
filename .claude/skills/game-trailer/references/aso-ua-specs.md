# ASO / UA 영상 규격 레퍼런스

플랫폼별 프로모 영상 정확한 요구사항 (2026년 기준).

---

## ASO — App Store (iOS)

| 항목 | 값 |
|------|-----|
| 최대 영상 수 | 3개 |
| 최대 길이 | 30초 |
| 포맷 | MP4, MOV |
| 코덱 | H.264, HEVC |
| 프레임레이트 | 30fps |
| 오디오 | AAC, 48kHz 이하 |

### iOS 해상도 (핵심)

| 디바이스 | 해상도 | 비율 | 비고 |
|----------|--------|------|------|
| iPhone 6.7" (필수) | **886×1920** | ~9:19.5 | Super Retina XDR, 모든 iPhone 자동 다운스케일 |
| iPhone 6.1" | 886×1920 호환 | - | 6.7" 제출하면 자동 적용 |
| iPad 12.9" | **1200×1600** | 3:4 | iPad 스토어 노출 시 별도 필요 |

> **핵심**: iOS는 **886×1920** 하나만 제출하면 모든 iPhone에 대응.
> 가로 게임은 1920×886으로 회전.

## FFmpeg 인코딩 설정

```bash
# 공통
-c:v libx264 -preset fast -movflags +faststart
-c:a aac -b:a 128k  # 오디오 있을 때만

# 품질별 CRF
low:    -crf 28   # 빠른 미리보기, AppLovin 4MB 대응
medium: -crf 23   # 일반 ASO/UA (기본)
high:   -crf 18   # 최종 배포 고품질
```

## 스케일링 전략

**원칙: 동일 방향 우선**. 세로 게임은 세로 포맷만, 가로 게임은 가로 포맷만 생성.
이렇게 하면 크롭이 최소화되고 레터박스/블러 등 보정이 불필요.

```bash
# 동일 방향: scale + crop (크롭 최소)
-vf "scale=W:H:force_original_aspect_ratio=increase,crop=W:H"

# 예: 400x600(2:3) → 1080x1920(9:16) = 약 11% 상하 크롭
# 예: 400x600(2:3) → 1080x1350(4:5) = 약 17% 상하 크롭
# 예: 1280x720(16:9) → 1920x1080(16:9) = 크롭 없음
```

크로스 방향 변환(세로→가로)은 기본적으로 하지 않음.
사용자가 명시적으로 요청하면 crop으로 처리 (중앙 크롭).
