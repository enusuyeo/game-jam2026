# TARTAROS 사운드 디렉션 가이드

## 1) 프로젝트 사운드 컨셉 요약
- 장르/톤: 다크 판타지 턴제 카드 RPG
- 배경: 타르타로스(지하 감옥) 탈출 서사
- 핵심 감정: 생존 압박, 유혹, 금기, 심판, 의지, 해방
- 핵심 테마: "강해지기 위해 타락할 것인가, 약해도 맹세를 지킬 것인가"

## 2) 글로벌 BGM 방향
- 메인 색채: Dark Orchestral + Ambient + Ritual Percussion
- 감정 곡선: 불안 -> 압박 -> 결의 -> 심판 -> 해방/허무
- 핵심 개선 방향: 단순 드론 중심 금지, "긴장감 있는 리듬 + 기억되는 짧은 멜로디 훅"을 항상 포함
- 공통 사운드 키워드:
  - subterranean, ritual, chained, forgotten, judgment, vow, hunger
- 권장 질감:
  - 저음 스트링 드론
  - 합창 패드(의식/신성 느낌)
  - 금속성 타격(쇠사슬, 철문 연상)
  - 둔탁한 심장박동형 킥
  - 4~8마디 리드 멜로디(스트링/브라스/벨 계열)

## 3) 스테이지별 사운드 컨셉

### Floor 1 - 낙인의 감방
- 무드: 음울, 경계, 초반 압박
- 연출 포인트: 차가운 감옥 공기, 첫 공포
- 권장 사운드: 저음 드론 + 중속 타악 + 차가운 벨 리드 훅

### Floor 2 - 균열의 저장고
- 무드: 유혹, 불안정, 리스크 선택
- 연출 포인트: 금지된 힘의 흔적, 대가의 암시
- 권장 사운드: 불협 스트링, 균열감 퍼커션, 짧은 오스티나토 멜로디

### Floor 3 - 망각의 복도
- 무드: 몽환, 상실, 혼란
- 연출 포인트: 기억 소거, 과거 회상
- 권장 사운드: 리버브 패드, 원거리 합창, 심박형 베이스 + 유령 같은 리드 멜로디

### Floor 4 - 심판의 현관
- 무드: 긴장 고조, 결의, 내적 충돌
- 연출 포인트: 최종 직전 시험, 의지 검증
- 권장 사운드: 점진적 빌드업, 강한 타악, 단호한 브라스 리프

### Floor 5 - 문지기의 제단
- 무드: 장엄, 공포, 최후 결단
- 연출 포인트: 최종 심판, 속박 vs 탈출
- 권장 사운드: 대규모 합창, 저음 타악, 강한 리드 테마 + 오케스트라 히트

## 4) 엔딩별 사운드 무드
- ENDING_LOOP: 승리 후 회귀, 허무와 냉기
- ENDING_TRUE: 무거운 해방, 숭고한 여운
- ENDING_FAIL: 의지 소거, 공허, 암전

## 5) 사운드 생성 시 필수 규칙
1. 이 문서를 최우선 기준으로 사용한다.
2. "멋진 음악"보다 "게임 서사와 감정선 일치"를 우선한다.
3. 과도한 현대 EDM/밝은 팝 톤은 금지한다.
4. 스테이지별로 같은 악기군을 유지하되, 긴장도만 단계적으로 상승시킨다.
5. 모든 BGM은 `25초` 길이의 루프 트랙으로 제작하고, 루프 경계가 들리지 않게 구성한다.
6. 사운드 생성 도구는 `codeb`만 사용한다. (다른 생성 툴 사용 금지)
7. 실제 게임 플레이를 기준으로, 너무 조용한 배경음악은 금지한다.
8. 전투 BGM은 항상 명확한 리듬 축(퍼커션/펄스)을 유지한다.
9. 모든 전투 BGM은 "짧고 선명한 멜로디 훅(4~8마디)"을 반드시 포함한다.
10. 템포는 기본적으로 `중속 이상(권장 110~150 BPM)`을 유지한다.
11. 패드/드론 단독 전개(리듬/멜로디 부재)는 금지한다.
12. Floor 1을 기준 테마(앵커)로 삼고, Floor 2~5는 같은 리드 모티프 변주로 제작한다.
13. 층이 바뀌어도 공통 악기군(저음 스트링, 의식 타악, 금속 타격, 다크 벨/플럭 리드, 저음 합창 패드)은 유지한다.

## 6) 층 번호 기반 배경 이미지 참조 규칙

  - Floor 2 -> `bg-02-*`

- 배경 파일에서 추출한 색감/질감 키워드를 사운드 톤 설계에 반드시 반영한다.

## 7) 재사용 프롬프트 (복붙용, codeb 전용)

```text
당신은 게임 오디오 디렉터입니다.
아래 문서를 단일 소스 오브 트루스로 삼아 사운드를 설계하세요:
/Users/oganesson/game-jam2026/design/sound-direction.md

작업 규칙:
1) 반드시 위 문서의 세계관/감정/스테이지 무드를 준수
2) 문서와 충돌하는 임의 해석 금지
3) 결과는 "25초 게임 BGM 루프용"으로 제작
4) 생성 도구는 codeb만 사용
5) 너무 조용한 결과 금지 (플레이 중 존재감 유지)
6) 해당 층 번호의 배경 이미지 색감/질감 반영
7) 전투용 리듬 축 + 4~8마디 멜로디 훅 필수
8) 템포는 110~150 BPM 범위 우선
9) 아래 출력 형식을 정확히 따를 것

이번 요청:
- 대상: [예: Floor 3 일반 전투 BGM]
- 길이: [고정: 25초 루프]
- 에너지: [예: 중간]
- 긴장도: [예: 중상]
- 변형 버전 수: [예: 2개]
- 배경 이미지: [예: /Users/oganesson/game-jam2026/assets/ui/pixel/background/bg-03-memory-mural-wall.png]

출력 형식:
1. 음악 컨셉 한 줄
2. 감정 키워드 5개
3. BPM 범위 / 조성(메이저/마이너/모달)
4. 핵심 악기 편성
5. 구조(도입-전개-클라이맥스-루프복귀)
6. 믹싱 지시(저역/중역/고역, 공간감)
7. 금지 요소 3개
8. 멜로디 훅 설명(리듬/음역/악기)
9. codeb 실행용 최종 프롬프트(바로 입력용)
10. codeb CLI 예시 명령 1개
```

## 8) codeb 생성 명령 템플릿

```bash
codeb cg audio sfx "<최종 프롬프트>" --duration 25 -o assets/sounds/generated/<file-name>.mp3
```
